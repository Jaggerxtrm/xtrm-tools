## Code Changes

### Worktree lifecycle hardening (xtrm-mt0r)

`cli/src/commands/end.ts`: Reordered cleanup — plugin unregister now runs only after successful `git worktree remove --force` + `git worktree prune --expire now`. Fallback `rmSync`, directory verification, and `clearStatuslineClaim` on success. Resolves main repo root via `git rev-parse --git-common-dir` (worktree-safe).

`cli/src/commands/worktree.ts`: Added `xt worktree doctor` reporting active/prunable/nested/orphan counts with remediation commands. Added `--orphans` and `--dry-run` flags to `xt worktree clean`. Successful removals also unregister plugin entries.

`cli/src/utils/worktree-session.ts`: Pre-flight guard blocks nested worktree creation — detects if cwd is inside a linked worktree, exits with actionable error pointing to `xt worktree doctor`.

`skills/xt-merge/SKILL.md` + `.claude/skills/xt-merge/SKILL.md`: Stage 3 updated — drop `--delete-branch`, explicit `git push origin --delete xt/<branch>` + non-fatal `git branch -d`. Both skill layers synced. `.specialists/default/xt-merge.specialist.yaml` updated to match.

### CLI destructive command confirmation (xtrm-78rf)

`cli/src/utils/confirmation.ts` (new): Centralizes `--yes`/TTY guard. `opts.yes → proceed`, no-TTY + no-`--yes` → safe fail with error, interactive TTY → `prompts()` confirm. All future destructive ops must use `confirmDestructiveAction()`.

Applied to: `reset.ts`, `end.ts`, `worktree.ts`, `clean.ts`, `install.ts`, `init.ts`, `claude.ts`, `pi.ts`, `merge.ts`.

### Statusline multi-claim (xtrm-lhn3)

`hooks/statusline.mjs`: Replaced single `claimId/claimTitle/claimStatus` with `claims[]` from `bd list --status=in_progress`. 0 claims → open count, 1 claim → id+title (max 40 chars), 2+ → id chips only. Bug fix: `^◐` line-start anchor + Set dedup eliminates priority-indicator false matches.

`config/pi/extensions/custom-footer/index.ts`: Same multi-claim logic, identical regex, same 3-tier rendering, removed session KV claim dependency.
