## Summary

Session delivered `ghgrep`, a zero-dependency Node.js CLI wrapping `mcp.grep.app` SSE transport, making GitHub code search available to any bash-capable specialist without requiring MCP tool access. Ships as a `bin` entry in the xtrm-tools package so it installs automatically with `npm install -g xtrm`. `ctx7` was simultaneously added to `machine-bootstrap` `MANAGED_DEPS` so `xt install` now ensures ctx7 alongside deepwiki and ghgrep. A `researcher` specialist (LOW perms, qwen3.5-plus) was created that orchestrates all three CLIs. A full `docs/bash-tools.md` reference was written covering ghgrep/ctx7/deepwiki with install source, flags, examples, and CLI-vs-MCP guidance. The second major thread was a deep design session on specialist persistent memory, sparked by analysis of Claude Code's internal memdir system. An overthinker ran two rounds and converged on a clean architecture: standalone `.specialists/memory/db.sqlite3` (runner-owned, decoupled from Dolt), typed schema, hybrid `<memory_write>` output block that routes to SQLite always and Dolt only when `project_share="true"`, FTS + weighted retrieval at startup (~1800 token budget), soft decay with `active → archived → superseded` lifecycle, scoped visibility (`private / role / project`) with three concrete role families, and wolf bootstrap on first init for cold-start. That design was filed as `unitAI-48pb` P3 in `~/projects/specialists`.
